# Restorevscodefiles
Recovering old files from the File History directory AppData

Данный файл позволит считать из системной папки Visal Code Studio автоматически сохраненые файлы. 

Случайно стер важные файлы проекта, узнал, что в папке c:\Users\ИМЯПОЛЬЗОВАТЕЛЯ\AppData\Roaming\Code\User\History\ хранятся копии файлов, попросил Claude Code создать панельку, которая считает json файлы и выведет в виде панели для возможности скачивания. 

Очень удобно, только нужно заменить путь в файле. 

## restore_files.php — VS Code History Browser

Утилита для просмотра и восстановления файлов из истории VS Code (`C:/Users/{user}/AppData/Roaming/Code/User/History`).

---

### Назначение

VS Code хранит историю изменений каждого файла в папке `History/` — по одной подпапке на файл с GUID-именем. Внутри каждой папки: `entries.json` (метаданные) и файлы версий с короткими ID. Без инструмента найти нужный файл по имени практически невозможно.

restore_files.php агрегирует все 10 000+ записей в единую таблицу с поиском, фильтрацией и возможностью скачать любую версию.

---

### Требования

- PHP 8.0+ (CLI или веб-сервер)
- Доступ с `localhost` — скрипт принимает запросы только от `127.0.0.1` / `::1`
- Путь к History захардкожен в константе `HISTORY_ROOT` (строка 14)

---

### Установка

1. Положить файл в webroot: restore_files.php
2. Открыть в браузере: `restore_files.php`

Если нужен другой пользователь Windows — изменить константу:

```php
const HISTORY_ROOT = 'C:/Users/ВАШ_ПОЛЬЗОВАТЕЛЬ/AppData/Roaming/Code/User/History';
```

---

### Функциональность

| Функция | Описание |
|---|---|
| **Поиск** | По имени файла или полному пути (подстрока) |
| **Фильтр по расширению** | Dropdown со всеми расширениями из истории |
| **Фильтр по дате** | Диапазон от/до |
| **Сортировка** | По любому столбцу (клик по заголовку), повторный клик — реверс |
| **Скачать версию** | Кнопка `⬇ Скачать` — отдаёт файл с оригинальным именем |
| **Подробнее** | Модальное окно с полным описанием изменения |

---

### Безопасность

- Запросы не с `127.0.0.1`/`::1` получают `403 Forbidden`
- При скачивании путь проверяется через `realpath()` + `strncmp` с `HISTORY_ROOT` — path traversal невозможен
- Имя скачиваемого файла санируется: `preg_replace('/[^\w\.\-]/', '_', $name)`

---

### Удаление после использования

Файл содержит путь к локальной системной папке пользователя и даёт доступ к истории кода. **После использования удалить** не использовать в публичном доступе.

```

    Qa3t.php
  5f5d1ab7/
    entries.json
    0xGa.js
```
